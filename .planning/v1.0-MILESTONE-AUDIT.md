### Partial Requirement

**AUDIT-4: Configurable digest byte order** — The option plumbing is complete from `WithDigestByteOrder()` through to `conn.SetDigestByteOrder()`, but the stored value is never retrieved by the pump layer. `ReadRawPDU` and `WriteRawPDU` are called without the `digestByteOrder` variadic argument, so they default to `binary.LittleEndian`. Setting `WithDigestByteOrder(binary.BigEndian)` has no effect on wire encoding.

**Impact:** Low for LIO targets (LittleEndian is default and correct). Would cause `DigestError` failures against targets that use BigEndian digest encoding.

**Fix:** Pass `conn.DigestByteOrder()` through `ReadPump`/`WritePump` to `ReadRawPDU`/`WriteRawPDU`.
