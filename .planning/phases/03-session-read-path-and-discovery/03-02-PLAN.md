    1. **Create internal/session/keepalive.go** (per D-08, SESS-05):
       Implements dual-direction NOP-Out/NOP-In keepalive per Pattern 4 from RESEARCH.md.

       - `(s *Session) keepaliveLoop(ctx context.Context)`:
         Background goroutine started by NewSession. Uses time.Ticker at s.cfg.keepaliveInterval (default 30s).
         On each tick:
         1. Register ITT via s.router.Register() (non-persistent, single response expected)
         2. Build NOP-Out PDU: Immediate=true, Final=true, InitiatorTaskTag=itt, TargetTransferTag=0xFFFFFFFF (initiator-originated ping). Set CmdSN from s.window.current() (carry but do NOT advance per Pitfall 3). Set ExpStatSN from s.getExpStatSN().
         3. Encode via pdu.EncodePDU, build RawPDU, send to s.writeCh
         4. Wait on Router channel with timeout (s.cfg.keepaliveTimeout, default 5s):
            - If NOP-In received: update ExpCmdSN/MaxCmdSN from response via s.window.update() and s.updateStatSN(nopin.StatSN). Success.
            - If timeout: call s.router.Unregister(itt), trigger session fatal error (connection presumed dead), return.
            - If ctx.Done(): cleanup and return.

       - `(s *Session) handleUnsolicitedNOPIn(raw *transport.RawPDU)`:
         Called from dispatchLoop when unsolicited NOP-In arrives (ITT=0xFFFFFFFF from unsolCh).
         Decode NOP-In from raw.BHS. Two sub-cases per Pitfall 5:
         - If nopin.TargetTransferTag != 0xFFFFFFFF: target-initiated ping. Respond with NOP-Out: Immediate=true, Final=true, ITT=0xFFFFFFFF (response, not new task), TTT=nopin.TargetTransferTag, CmdSN=s.window.current(), ExpStatSN=s.getExpStatSN(). Encode and send to writeCh. Also echo any data from the NOP-In.
         - If nopin.TargetTransferTag == 0xFFFFFFFF: informational. Just update ExpCmdSN/MaxCmdSN.
         - In both cases: call s.window.update(nopin.ExpCmdSN, nopin.MaxCmdSN) and s.updateStatSN(nopin.StatSN).

    2. **Create internal/session/async.go** (per D-09, D-10, EVT-01):
       Handles AsyncMsg PDUs from the target.

       - `(s *Session) handleAsyncMsg(raw *transport.RawPDU)`:
         Decode AsyncMsg from raw.BHS + raw.DataSegment.
         Update sequence numbers: s.window.update(async.ExpCmdSN, async.MaxCmdSN), s.updateStatSN(async.StatSN).
         Build AsyncEvent from AsyncMsg fields: EventCode, AsyncVCode (as VendorCode), Parameter1/2/3, Data=raw.DataSegment.

         Special handling for EventCode per RFC 7143 Section 11.9:
         - EventCode 0 (SCSI async event): dispatch to asyncHandler with sense data
         - EventCode 1 (target requests logout): per D-10, initiate graceful logout automatically. Call s.handleTargetRequestedLogout(async) in a goroutine. Then dispatch to asyncHandler.
         - EventCode 2 (connection drop): dispatch to asyncHandler, set session error
         - EventCode 3 (session drop): dispatch to asyncHandler, set session error, close
         - EventCode 4 (negotiation request): dispatch to asyncHandler (renegotiation is Phase 6+)
         - EventCode 255 (vendor-specific): dispatch to asyncHandler
         - All others: dispatch to asyncHandler

         If s.cfg.asyncHandler is nil, log at slog.Warn level and discard.

       - `(s *Session) handleTargetRequestedLogout(evt AsyncEvent)`:
         Per D-10: wait DefaultTime2Wait seconds (from s.params.DefaultTime2Wait), then initiate logout. Call s.logout(context.Background(), 0) (reason=0, close session). After logout completes, close session.

    3. **Update internal/session/session.go dispatchLoop**:
       Wire the unsolicited PDU handling. The dispatchLoop should check opcode of unsolicited PDUs:
       - OpNOPIn: call s.handleUnsolicitedNOPIn(raw)
       - OpAsyncMsg: call s.handleAsyncMsg(raw)
       - Other: log warning via slog

    1. **Create internal/session/logout.go** (per EVT-02, EVT-03):
       Implements graceful logout per RFC 7143 Section 11.14/11.15.

       - `(s *Session) logout(ctx context.Context, reasonCode uint8) error`:
         Private method performing the Logout PDU exchange:
         1. Acquire CmdSN from window (Logout is a non-immediate command that advances CmdSN)
         2. Register ITT via s.router.Register() (single response expected)
         3. Build LogoutReq PDU: Final=true, ReasonCode=reasonCode (0=close session, 1=close connection, 2=remove connection for recovery per EVT-03), CID=0 (single connection), CmdSN=acquired, ExpStatSN=s.getExpStatSN(), ITT=itt
         4. Encode, build RawPDU, send to writeCh
         5. Wait on Router channel with timeout (DefaultTime2Wait + DefaultTime2Retain seconds from params, or 30s max):
            - If LogoutResp received: decode, check Response field (0=success). Update sequence numbers. Return nil on success, error on non-zero response.
            - If timeout: return error (target did not respond to logout)
            - If ctx.Done(): cleanup, return ctx.Err()

       - `(s *Session) Logout(ctx context.Context) error`:
         Public method for graceful session logout. Per Pitfall 7 from RESEARCH.md:
         1. Set session state to "logging out" (prevent new Submit calls -- Submit should check state and return error)
         2. Wait for all in-flight tasks to complete (check s.tasks map is empty, or timeout after reasonable duration like 30s). Use a ticker to poll s.tasks length under s.mu.
         3. Call s.logout(ctx, 0) for session close
         4. On success, close session internals (cancel context, stop pumps)
         5. Return error if any step fails

       - `(s *Session) LogoutConnection(ctx context.Context, reasonCode uint8) error`:
         For EVT-03 (connection recovery logout). Calls s.logout(ctx, reasonCode). ReasonCode 2 = remove connection for recovery.

    2. **Integrate with Session.Close()**:
       Update Session.Close() in session.go (if not already integrated):
       - If session is in logged-in state, attempt Logout first with a timeout context (5s)
       - Then close underlying connection and stop goroutines
       - Close is idempotent via sync.Once
