**DigestError type with CRC32C verification in ReadRawPDU plus compact String() methods on all 18 PDU types for wire-level debugging**

## Accomplishments
- DigestError returned on CRC32C mismatch in ReadRawPDU for both header and data digests, extractable via errors.As
- All 18 PDU types implement fmt.Stringer with compact key:value format using hex for tags and decimal for sequence numbers
- truncHex helper limits CDB display to first 8 bytes in colon-separated hex

## Decisions Made
- DigestError uses pointer receiver so errors.As works correctly with interface matching
- Stringer format uses lowercase hex for ITT/TTT tags, uppercase hex for status/reason bytes
- CDB truncated to first 8 bytes since full 16 bytes is noisy in logs; 8 bytes covers the opcode + key parameters
