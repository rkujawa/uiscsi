    ```
    func Login(ctx context.Context, tc *transport.Conn, opts ...LoginOption) (*NegotiatedParams, error)
    ```
    - Apply options to loginConfig with defaults: sessionType="Normal", initiatorName="iqn.2026-03.com.github.rkujawa:uiscsi", headerDigest=["None"], dataDigest=["None"]
    - If isid is zero-value, generate random 6-byte ISID using crypto/rand (ISID format: first byte = 0x40 for random type per RFC 7143 Section 11.12.5)
    - Determine auth method: if chapUser != "" -> "CHAP", else "None" (per D-07)
    - Create loginState and call state machine loop
    - On success: call tc.SetDigests(params.HeaderDigest, params.DataDigest) and tc.SetMaxRecvDSL(params.MaxRecvDataSegmentLength) (Pitfall 6: only after login completes)

    **Login state machine:**
    ```
    type loginState struct {
        conn       net.Conn          // raw TCP, not pumps (Pitfall 5)
        cfg        *loginConfig
        params     *NegotiatedParams
        chap       *chapState        // nil if AuthMethod=None
        csg        uint8             // current stage
        cmdSN      uint32            // set once, not incremented during login (Pitfall 10)
        expStatSN  uint32            // tracks target's StatSN (Pitfall 9)
        tsih       uint16            // 0 for new session
        isid       [6]byte
    }
    ```

    State machine loop (`func (ls *loginState) run(ctx context.Context) error`):
    1. **Stage 0 (SecurityNegotiation):**
       - First PDU: send InitiatorName, TargetName, SessionType, AuthMethod (CHAP or None)
       - If AuthMethod=None: set Transit=true, NSG=1 (to operational negotiation)
       - If AuthMethod=CHAP: send CHAP_A=5, set Transit=false, NSG=0
       - Read response, check StatusClass != 0 -> return &LoginError
       - If CHAP: process CHAP exchange via chapState.processChallenge
         - Send CHAP_N + CHAP_R (+ initiator CHAP_I/CHAP_C if mutual)
         - Read response, if mutual -> chapState.verifyMutualResponse
       - After security done: Transit=true, CSG=0, NSG=1

    2. **Stage 1 (LoginOperationalNegotiation):**
       - Build operational keys via buildInitiatorKeys(cfg)
       - Send with Transit=true, CSG=1, NSG=3
       - Read response, resolve each key via resolveKey against keyRegistry
       - Apply resolved keys via applyNegotiatedKeys

    3. **Stage 3 (FullFeaturePhase):**
       - When response has Transit=true, NSG=3 -> login complete
       - Store TSIH from response in params
       - Return nil (success)

    PDU send helper (`func (ls *loginState) sendLogin(ctx context.Context, keys []KeyValue, transit bool, csg, nsg uint8) (*pdu.LoginResp, error)`):
    - Build LoginReq with fields: Transit=transit, CSG=csg, NSG=nsg, VersionMax=0x00, VersionMin=0x00, ISID=ls.isid, TSIH=ls.tsih, CmdSN=ls.cmdSN, ExpStatSN=ls.expStatSN, Data=EncodeTextKV(keys)
    - Encode via pdu.EncodePDU, write via transport.WriteRawPDU to ls.conn directly (NOT via WritePump -- Pitfall 5)
    - Set deadline on conn from ctx
    - Read response via transport.ReadRawPDU(ls.conn, false, false) -- no digests during login (Pitfall 6)
    - Decode LoginResp via UnmarshalBHS, set Data from RawPDU.DataSegment
    - Update ls.expStatSN from response (Pitfall 9): expStatSN = response.Header.StatSN + 1 (read StatSN from BHS bytes 24-27)
    - Do NOT increment CmdSN (Pitfall 10)
    - Return LoginResp

    Important implementation notes:
    - StatSN is in BHS bytes 24-27 of LoginResp. The Header struct has a StatSN field -- read it from there after unmarshal. If Header.StatSN is not exposed, read it directly from raw BHS bytes using binary.BigEndian.Uint32(bhs[24:28]).
    - Login PDUs use ITT for correlation but during login we do synchronous request-response, so ITT can be a fixed value (e.g., 0).
    - The Continue bit handling: if a response has Continue=true, accumulate data segments until Continue=false before processing keys. For the initial implementation, support this but most targets send single-PDU responses.

    `func runMockTarget(t *testing.T, ln net.Listener, cfg mockTargetConfig)`:
    - Accept one connection, defer close
    - Loop: ReadRawPDU from connection (no digests during login)
    - Decode LoginReq from BHS, decode text keys from DataSegment
    - Track stages via CSG/NSG in incoming PDUs
    - **Stage 0 (Security):**
      - If cfg.authMethod == "None": respond with AuthMethod=None, Transit=true, CSG=0, NSG=1
      - If cfg.authMethod == "CHAP":
        - First request (has AuthMethod=CHAP, CHAP_A=5): respond with CHAP_A=5, CHAP_I=<random_id>, CHAP_C=<random_challenge_hex>
        - Second request (has CHAP_N, CHAP_R): verify CHAP_R using chapResponse(id, secret, challenge). If valid and cfg has mutual: respond with target's CHAP_N + CHAP_R. Set Transit=true, NSG=1.
        - If CHAP_R invalid: respond with StatusClass=2, StatusDetail=1 (AuthFailure)
    - **Stage 1 (Operational):**
      - Parse initiator's proposed keys
      - Respond with target's values: HeaderDigest=cfg.headerDigest, DataDigest=cfg.dataDigest, MaxBurstLength=cfg.maxBurstLength (or default), etc.
      - Set Transit=true, CSG=1, NSG=3, TSIH=cfg.tsih
    - If cfg.statusClass != 0: send error response at appropriate point
    - Track StatSN starting at 0, increment per response
