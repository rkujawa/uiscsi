Note on AbortTask: AbortTask requires an active task tag to abort. In a synchronous test, all tasks complete before we can abort them. Testing AbortTask meaningfully against a real target requires a long-running command. Include a basic AbortTask test that sends the TMF even if the task has already completed -- the target should respond with "task not found" or similar, which validates the TMF path works end-to-end. If the public API does not expose a way to get pending task tags, skip AbortTask and document why.

**test/e2e/recovery_test.go** -- Error recovery / connection drop (D-06 scenario 7):

`func TestErrorRecovery_ConnectionDrop(t *testing.T)`:
1. RequireRoot + RequireModules
2. Setup LIO target with default LUN, TargetSuffix "recovery"
3. defer cleanup()
4. Dial with `uiscsi.WithTarget(...)`, `uiscsi.WithInitiatorName(...)`, `uiscsi.WithMaxReconnectAttempts(3)`
5. Verify session is functional: `sess.Inquiry(ctx, 0)` succeeds
6. **Connection drop strategy** (Claude's discretion -- use TCP proxy approach from RESEARCH.md):
   - The cleanest approach per RESEARCH.md is a simple TCP proxy. However, if the uiscsi public API doesn't expose the underlying connection, an alternative is to remove and recreate the LIO network portal to force-close the TCP connection from the target side:
     a. Remove NP directory: `os.Remove(npPath)` -- this kills the TCP listener and all active connections
     b. Wait briefly (100-500ms) for the library to detect the drop
     c. Recreate NP: `os.MkdirAll(npPath, 0o755)` -- new listener ready for reconnect
   - If the Target struct from lio.Setup exposes NP path info, use it. Otherwise, construct the path from the known IQN and port.
   - Alternative simpler approach: use `ss -K` to kill the specific TCP socket (requires root, which we have). Command: `exec.Command("ss", "-K", "src", "127.0.0.1", "sport", "=", strconv.Itoa(tgt.Port))`.
7. After drop + recreate, try `sess.Inquiry(ctx, 0)` -- should succeed after ERL 0 reconnect
8. Use a generous timeout (60s context) since reconnect may take time
9. Close session

Use 30s context timeouts for multi-LUN and TMF tests. Use 60s for recovery test.
