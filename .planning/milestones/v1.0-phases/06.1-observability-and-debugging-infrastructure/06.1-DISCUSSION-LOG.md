## Digest Verification Strategy

| Option | Description | Selected |
|--------|-------------|----------|
| Strict RFC — fail immediately | ReadRawPDU verifies inline, returns DigestError, connection-fatal | ✓ |
| Verify-and-report, caller decides | Returns PDU + verification result, caller chooses action | |
| Separate verification step | VerifyDigests() function called explicitly by caller | |

**User's choice:** Strict RFC — fail connection immediately on mismatch
**Notes:** RFC 7143 Section 7.3 mandates connection error for ERL 0. DigestError carries expected/actual/which-digest for diagnostics.

## PDU Tracing Architecture

| Option | Description | Selected |
|--------|-------------|----------|
| Callback hooks on Session | WithPDUHook(func(direction, *RawPDU)) matching WithAsyncHandler pattern | ✓ |
| io.ReadWriter middleware | Wrap net.Conn with tracing layer, sees raw bytes | |
| slog-only approach | Structured slog records at Debug level, no hooks API | |
| Both hooks + slog | Hooks for programmatic test access, slog for human traces | |

**User's choice:** Callback hooks on Session
**Notes:** Matches existing WithAsyncHandler pattern. Clean, testable, no new abstraction layer.

---

## Metrics Interface Design

| Option | Description | Selected |
|--------|-------------|----------|
| Concrete stats struct | Session.Stats() returns snapshot with atomic counters | |
| Callback-based metrics sink | WithMetricsHook(func(MetricEvent)) push-based | ✓ |
| slog attributes only | Emit metrics as slog fields on existing log lines | |
| Stats struct + optional callback | Both polling and push-based | |

**User's choice:** Callback-based metrics sink
**Notes:** Push-based keeps library dependency-free. Consumers wire to Prometheus/OpenTelemetry/whatever.

---

## Logging Granularity

| Option | Description | Selected |
|--------|-------------|----------|
| Debug=every PDU, Info=state, Warn=recoverable | Full PDU trace at Debug, login/session at Info, digest/timeout at Warn | ✓ |
| Debug=state, Info=lifecycle only | Conservative, PDU tracing left to hooks | |
| Two debug tiers via slog groups | Debug + "pdu" group for fine-grained control | |

**User's choice:** Debug = every PDU, Info = state changes, Warn = recoverable errors
**Notes:** Clear separation. Error reserved for unrecoverable failures only.

## Additional Decisions

### PDU String() Format
- Confirmed: `StructName{Key:Value Key:Value ...}` format
- Example: `SCSICommand{OpCode:0x01 ITT:0x0001 CmdSN:42 LUN:0 CDB:28:00:...}`
- Hex for opcodes/ITTs, decimal for sequence numbers
