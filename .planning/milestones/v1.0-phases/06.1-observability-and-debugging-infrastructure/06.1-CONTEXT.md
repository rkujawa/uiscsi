# Phase 06.1: Observability and Debugging Infrastructure - Context

**Gathered:** 2026-04-01
**Status:** Ready for planning

<domain>
## Phase Boundary

Fill all observability gaps so E2E tests can diagnose protocol compliance issues from logs alone. This phase adds digest verification, PDU tracing, structured logging, metrics, and String() methods — it does NOT add new protocol features, change existing APIs, or implement E2E test infrastructure (that's Phase 7).

</domain>

<decisions>
## Implementation Decisions

### Digest Verification
- **D-01:** Strict RFC 7143 Section 7.3 compliance — fail connection immediately on CRC32C mismatch
- **D-02:** `ReadRawPDU` verifies digests inline when digest flags are set; returns typed `DigestError` with expected value, actual value, and which digest (header/data)
- **D-03:** ReadPump treats `DigestError` as connection-fatal (ERL 0 behavior). SNACK-based recovery for ERL 1+ can use the error type to distinguish digest failures from other read errors

### PDU Tracing
- **D-04:** Callback hooks via `WithPDUHook(func(direction, *RawPDU))` session option, matching existing `WithAsyncHandler` pattern
- **D-05:** Hook is called in ReadPump (after read, before dispatch) and WritePump (before write) so tests can record full PDU sequences
- **D-06:** Hook sees `*RawPDU` (raw wire bytes) — callers decode if they need typed fields

### PDU String Methods
- **D-07:** All 18 PDU types get `String()` methods returning key fields in compact format: `SCSICommand{OpCode:0x01 ITT:0x0001 CmdSN:42 LUN:0 CDB:28:00:...}`
- **D-08:** Format is struct-name + curly braces with space-separated key:value pairs. Hex for opcodes/ITTs, decimal for sequence numbers, truncated hex for CDB/data

### Metrics Interface
- **D-09:** Push-based callback via `WithMetricsHook(func(MetricEvent))` session option
- **D-10:** `MetricEvent` is a typed enum covering: PDUSent, PDUReceived (with opcode), CommandComplete (with latency), BytesIn, BytesOut
- **D-11:** No concrete stats struct — consumers aggregate as they see fit. Library stays dependency-free

### Logging Granularity
- **D-12:** Debug = every PDU send/receive with opcode, ITT, CmdSN, StatSN, data lengths
- **D-13:** Info = login stage transitions, session open/close, reconnection start/complete, command window changes
- **D-14:** Warn = digest mismatches (before connection drop), dropped PDUs, timeout retries
- **D-15:** Error = unrecoverable failures only
- **D-16:** All slog calls use the session's injected `*slog.Logger` (via `WithLogger`), never the global `slog` package directly — except in transport/pump.go which has no session context (convert to accept logger parameter)

### Claude's Discretion
- Exact `MetricEvent` type design (interface vs struct with type field vs sum type)
- slog attribute naming conventions (camelCase vs snake_case)
- Whether `String()` methods go on the typed PDU structs or on `RawPDU` (typed structs preferred)
- Internal helper organization (new files vs extending existing)

</decisions>

## Existing Code Insights

### Reusable Assets
- `OpCode.String()` in `internal/pdu/opcode.go` — established pattern for human-readable output
- `WithAsyncHandler(func(AsyncEvent))` in `internal/session/types.go` — established pattern for callback hooks
- `WithLogger(*slog.Logger)` in `internal/session/types.go` — logger injection already plumbed, just underused
- `HeaderDigest()` / `DataDigest()` in `internal/digest/crc32c.go` — computation ready, just needs verification call
- `RawPDU` in `internal/transport/framer.go` — has all wire bytes including digest fields

### Established Patterns
- Functional options pattern (`SessionOption`) for all session configuration
- `DecodeBHS()` dispatches to typed PDU structs — `String()` goes on these typed structs
- Transport layer uses `io.Reader`/`io.Writer` — hooks must work at RawPDU level, not raw bytes
- Buffer pooling in transport (`GetBHS`, `GetBuffer`, `PutBuffer`) — don't allocate in hot path logging

### Integration Points
- `ReadRawPDU` — add digest verification after reading, before returning
- `ReadPump` — add PDU hook call after successful read, add slog.Debug per PDU
- `WritePump` — add PDU hook call before write, add slog.Debug per PDU
- `Session` — new `WithPDUHook` and `WithMetricsHook` options, plumb to transport layer
- `pump.go` lines 62-63 and 69-70 — convert bare `slog.Warn` to use injected logger

</code_context>
