Create `uiscsi-ls/probe.go` (package main):

1. Define a `discoverer` interface to enable test stubbing of uiscsi calls:
```go
type discoverer interface {
    Discover(ctx context.Context, addr string, opts ...uiscsi.Option) ([]uiscsi.Target, error)

```
Use a package-level variable `var discoverFunc = uiscsi.Discover` so tests can swap it. Alternatively, define the interface and accept it as a parameter to `probePortal`. Choose whichever keeps the code simple -- the key requirement is that `probePortal`'s call to `uiscsi.Discover` can be replaced in tests.

2. `func normalizePortal(addr string) string` -- per D-06:
   - Use `net.SplitHostPort(addr)`. If error (no port), return `net.JoinHostPort(addr, "3260")`.
   - If port is empty string, default to "3260".
   - Return `net.JoinHostPort(host, port)`.
   - Handle IPv6 bracket syntax correctly via net.SplitHostPort/JoinHostPort.

3. `func resolveCHAP(flagUser, flagSecret string) (string, string)` -- per D-05:
   - `user := flagUser; if user == "" { user = os.Getenv("ISCSI_CHAP_USER") }`
   - `secret := flagSecret; if secret == "" { secret = os.Getenv("ISCSI_CHAP_SECRET") }`
   - Return `(user, secret)`

4. `func probeAll(ctx context.Context, portals []string, opts []uiscsi.Option) []PortalResult`:
   - Iterate over portals sequentially (parallel would complicate error handling for v1).
   - For each portal, call `probePortal(ctx, portal, opts)` and append result.
   - Return all results.

5. `func probePortal(ctx context.Context, portal string, opts []uiscsi.Option) PortalResult`:
   - Create a per-portal context with 10-second timeout: `pCtx, cancel := context.WithTimeout(ctx, 10*time.Second); defer cancel()`
   - Call `uiscsi.Discover(pCtx, portal, opts...)` (or via the stub mechanism) to get targets. On error, return `PortalResult{Portal: portal, Err: err}`.
   - For each target:
     - Call `uiscsi.Dial(pCtx, portal, append(opts, uiscsi.WithTarget(t.Name))...)`. On error, record `TargetResult{IQN: t.Name, Err: err}` and continue to next target.
     - Call `sess.ReportLuns(pCtx)`. On error, record error and `sess.Close()` and continue.
     - For each LUN:
       - Call `sess.Inquiry(pCtx, lun)`. On error, create LUNResult with what is known and continue.
       - Build `LUNResult` with DeviceType, DeviceTypeS (from `deviceTypeName`), Vendor (from `strings.TrimSpace(inq.VendorID)`), Product (from `strings.TrimSpace(inq.ProductID)`), Revision (from `strings.TrimSpace(inq.Revision)`).
       - Per Pitfall 3 from RESEARCH.md: Only call `sess.ReadCapacity(pCtx, lun)` if DeviceType is 0x00 or 0x0E (disk types). For other types, set CapacityStr to "-".
       - On ReadCapacity success: set CapacityBytes, BlockSize, LogicalBlocks, and CapacityStr from `formatCapacity(cap.LogicalBlocks, cap.BlockSize)`. On error: set CapacityStr to "-".
     - `sess.Close()`
   - Return assembled PortalResult.

Imports needed: `context`, `fmt`, `net`, `os`, `strings`, `time`, `github.com/rkujawa/uiscsi`.
