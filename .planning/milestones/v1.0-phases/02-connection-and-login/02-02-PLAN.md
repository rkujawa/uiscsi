<objective>
Implement CHAP (Challenge Handshake Authentication Protocol) for iSCSI login, covering both one-way and mutual authentication.

Purpose: CHAP is the authentication mechanism specified by RFC 7143. One-way CHAP lets the target authenticate the initiator; mutual CHAP adds bidirectional verification. This is pure crypto logic with no network I/O -- the login state machine (Plan 03) drives the PDU exchange.

Output: `internal/login/chap.go` with CHAP response computation, binary value encoding, and exchange state tracking.
</objective>

  <action>
    **chap.go**:

    Import: `crypto/md5`, `crypto/rand`, `encoding/hex`, `fmt`, `strconv`, `strings`, `encoding/base64`

    Core functions:
    - `func chapResponse(id byte, secret, challenge []byte) [16]byte` -- computes MD5(single_byte_id || secret_bytes || challenge_bytes). Uses md5.New(), writes id as single byte, then secret, then challenge. Returns [16]byte. Critical: id is ONE byte, not a multi-byte integer (Pitfall 1).
    - `func encodeCHAPBinary(data []byte) string` -- returns "0x" + hex.EncodeToString(data). Always lowercase hex.
    - `func decodeCHAPBinary(s string) ([]byte, error)` -- if s starts with "0x" or "0X", hex.DecodeString(s[2:]). If s starts with "0b" or "0B", base64.StdEncoding.DecodeString(s[2:]). Otherwise return error.

    CHAP exchange state:
    - `type chapState struct` with fields: user string, secret string, mutualSecret string (for mutual CHAP), isMutual bool, initiatorID byte, initiatorChallenge []byte
    - `func newCHAPState(user, secret string, mutual bool, mutualSecret string) *chapState` -- sets fields. If mutual, generates random initiatorID (single byte from crypto/rand) and random initiatorChallenge (16 bytes from crypto/rand).
    - `func (cs *chapState) processChallenge(keys map[string]string) (map[string]string, error)` -- called when target sends CHAP_A, CHAP_I, CHAP_C:
      1. Parse CHAP_A, verify it's "5" (MD5). Return error if unsupported algorithm.
      2. Parse CHAP_I as decimal integer, take low byte as id.
      3. Decode CHAP_C via decodeCHAPBinary to get challenge bytes.
      4. Compute response = chapResponse(id, secret, challenge).
      5. Build response keys: CHAP_N=cs.user, CHAP_R=encodeCHAPBinary(response[:])
      6. If mutual: also add CHAP_I=strconv.Itoa(int(cs.initiatorID)), CHAP_C=encodeCHAPBinary(cs.initiatorChallenge)
      7. Return response keys map.
    - `func (cs *chapState) verifyMutualResponse(keys map[string]string) error` -- called for mutual CHAP when target responds with its own CHAP_N, CHAP_R:
      1. Parse CHAP_R, decode via decodeCHAPBinary.
      2. Compute expected = chapResponse(cs.initiatorID, []byte(cs.mutualSecret), cs.initiatorChallenge).
      3. Compare with constant-time comparison (crypto/subtle.ConstantTimeCompare). Return error if mismatch.

    All functions are package-private (lowercase) since only the login state machine calls them.
