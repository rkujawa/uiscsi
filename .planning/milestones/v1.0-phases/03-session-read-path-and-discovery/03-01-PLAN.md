<objective>
Build the core session layer: types, CmdSN command windowing, SCSI command dispatch via async Submit+Channel, and Data-In reassembly into io.Reader streams.

Purpose: This is the foundation for all session-level operations. The session wraps a transport.Conn after login, provides CmdSN/MaxCmdSN flow control per RFC 7143, dispatches SCSI commands with ITT correlation, and reassembles multi-PDU Data-In responses with DataSN validation. Keepalive, async events, logout, and discovery build on this in subsequent plans.

Output: `internal/session/` package with Session struct, Submit method, command windowing, and Data-In reassembly.

       - `(s *Session) dispatchLoop(ctx context.Context)`:
         Per Pattern 6 from RESEARCH.md, the session intercepts ALL PDUs before Router:
         Read from unsolCh for unsolicited PDUs (ITT=0xFFFFFFFF).
         BUT for solicited PDUs (Data-In, SCSIResponse), ReadPump currently dispatches via Router.
         **Key design decision:** Since Router.Dispatch deletes entry after one PDU (Pitfall 1), the session layer needs a different approach for multi-PDU commands. Two options:
         (a) Modify Router to support persistent registrations -- add `RegisterPersistent(itt) chan *RawPDU` that doesn't auto-delete on Dispatch.
         (b) Have session register with Router using a single dispatch channel per task, and NOT delete on dispatch.

         **Use approach (a):** Add `RegisterWithITT(itt uint32, ch chan<- *RawPDU)` to Router that registers a specific ITT with a provided channel (does NOT auto-delete on Dispatch). The session calls this for each submitted command. When the task completes (S-bit or SCSIResponse), session calls Router.Unregister(itt).

         Actually, to minimize changes to Router and avoid breaking existing tests, use a **session-level pduCh**: modify the ReadPump call to use a session-level Router where Dispatch does NOT delete entries. Add a `DispatchKeep(itt uint32, pdu *RawPDU) bool` method to Router that delivers but does not delete. Session uses DispatchKeep in its own copy of the dispatch logic.

         **Simplest approach:** Add `RegisterPersistent(itt uint32) (<-chan *RawPDU)` to Router. This creates a buffered channel (cap 8) and stores it. Dispatch delivers to it but does NOT delete the entry. Unregister removes it. This way ReadPump's existing dispatch logic works unchanged.

         Add to internal/transport/router.go:
         - `RegisterPersistent(itt uint32) <-chan *RawPDU`: like Register but the pending entry has a `persistent` flag. Dispatch checks the flag and skips delete if true.
         - Modify Dispatch: `if ok && !entry.persistent { delete(...) }`
         - Change `pending map[uint32]chan<- *RawPDU` to `pending map[uint32]*routerEntry` where `routerEntry` is `{ ch chan<- *RawPDU, persistent bool }`. Update Register, Dispatch, Unregister to use entry struct.

         Then in dispatchLoop, handle unsolicited PDUs from unsolCh:
         - NOPIn with TTT != 0xFFFFFFFF: respond with NOP-Out echoing TTT (per Pitfall 5). NOPIn with TTT = 0xFFFFFFFF: just update ExpCmdSN/MaxCmdSN.
         - AsyncMsg: defer to plan 02 (store handler, call if set)
         - Other unsolicited: log warning

         For solicited PDUs, each task's channel receives PDUs via Router (now persistent). A per-task goroutine drains the channel:
         - DataIn: call task.handleDataIn
         - SCSIResponse: call task.handleSCSIResponse, then Unregister ITT and remove from tasks map
         This per-task goroutine prevents one slow reader from blocking other tasks (Open Question 2).

       - `(s *Session) updateStatSN(statSN uint32)`: under s.mu, if serial.GreaterThan or equal, update expStatSN = statSN + 1. Per SESS-03.

       - `(s *Session) getExpStatSN() uint32`: returns current expStatSN under s.mu

       - `(s *Session) Close() error`: stop accepting new commands, close cmdWindow, wait for in-flight tasks to complete or timeout, close writeCh, cancel context, wait for done. Return first error.

       - `(s *Session) Err() error`: returns the session-fatal error if any
